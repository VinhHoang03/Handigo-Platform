# Status Workflow

Use this workflow to summarize known task state, active work, and validation status.

## Goal

Answer "where are we now?" without re-reading the entire project.

## Include When Relevant

- current task or feature area
- changed files or major touched areas
- validation state
- preview status
- pending work or known blockers

## Process

1. Do not run Git commands unless the user explicitly asks for Git status/history/diff.
2. Inspect only the files, command outputs, or task context that are directly relevant.
3. Summarize the active workstream.
4. Report validation that has and has not been run.
5. Include preview or deployment state if the task depends on it.
6. End with the most relevant next action.
